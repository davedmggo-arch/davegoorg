In-Class Topics

  • Learn how to navigate Blender’s 3D viewport (orbit, pan, zoom) and understand why studios favor a mouse for precision.
  • Understand Object Mode for whole-object manipulation and practice creating a correctly scaled tabletop.
  • Explore Edit Mode for geometry-level editing with vertex, edge, and face selections.
  • Apply the four core modeling maneuvers: loop cuts, bevels, insets, and extrusions, to build functional table legs.
  • Research and design a distinct type of table to give the project purpose, style, and real-world believability.

Key Concepts & Tools


Introduction
You already know Grease Pencil’s 2D navigation and strokes. In 3D, the big shift is:
  • You’re working with geometry, volume, and scale, not just lines.
  • You now move around objects (orbit), not just across a flat canvas.

Objective: By the end of class, each student will model a unique, functional table (coffee table, dining table, nightstand, drafting desk, etc.) with correct scale.

In design and visual development, researching reference is as important as the modeling itself. Different tables have different proportions, structures, and purposes. Before diving in, think:
  • Is it for eating, studying, display, children’s use, industrial work?
  • How tall should it be? How thick should the legs be? What’s its style (minimalist, ornate, rustic, modern)?
This ensures your table is distinct and purposeful, not just a default cube stretched thin.
Navigation in 3D
Blender's navigational tools:
  • 3D Viewport: main window where you model.
  • Gizmos (top-right corner): orbit, pan, zoom, axis widget.
  • View menu: camera/viewpoint controls.
  • Tilde (~) hotkey: quick view pie.
  • N-panel → View tab: camera lens, clip range, lock options.

Key concepts
  • Orbit: rotate around the focus object. (New compared to GP).
  • Pan: slide camera sideways. (Similar to GP panning).
  • Zoom: dolly in/out. (Similar to GP zoom).

Controls by hardware
Mouse (studio standard):
  • Orbit: MMB drag
  • Pan: Shift + MMB
  • Zoom: Scroll wheel or Ctrl + MMB drag
  • Frame selected: Numpad .

Emulate 3-button (laptop/stylus users):
  • Orbit: Alt + LMB
  • Pan: Shift + Alt + LMB
  • Zoom: Ctrl + Alt + LMB

Stylus (tablet): map one button to MMB, another to RMB. Works, but less reliable for production.

Why studios prefer a mouse: More precise, less buggy, universally supported in VFX pipelines, and all training/docs assume MMB. Tablets are excellent for sculpting or painting, but not for day-to-day modeling/navigation.
Orthographic Views
What they are:
  • Orthographic views flatten perspective to remove distortion.
  • Essential for accurate modeling, aligning geometry, and working to real-world measurements.
  • Think of them as “blueprint views”: front, side, and top.

Why they matter:
In Perspective View, parallel lines converge (just like in photography). This can distort proportions and make measurements deceptive.
Orthographic Views give you undistorted side profiles so you can:
  • Ensure legs are evenly aligned.
  • Check proportions and symmetry.
  • Accurately position elements like tabletop thickness and leg spacing.

Accessing orthographic views:
Numpad keys:
  • 1 = Front view
  • Ctrl + 1 = Back view
  • 3 = Right view
  • Ctrl + 3 = Left view
  • 7 = Top view
  • Ctrl + 7 = Bottom view
Toggle Perspective/Orthographic: Numpad 5.
Tilde (~) key (View Pie menu): quick access to Front, Right, Top, Camera, Walk/Fly navigation.

Workflow tip:
  • Start in Perspective View for creativity and general shape.
  • Switch to Orthographic Views to lock in accuracy, proportions, and alignment.
Object Mode Basics
What is Object Mode?
  • Works on entire objects as units.
  • You can: Move, Rotate, Scale, Duplicate, Parent, Apply transforms, Add modifiers.
  • Compare to Grease Pencil layers: each object is separate and managed in the Outliner.

Core transforms
  • G = Move
  • R = Rotate
  • S = Scale
  • Right-Click/Escape = Cancel transformation
  • Constrain with X/Y/Z.
  • Lock with Shift + X/Y/Z
  • Apply transforms with Ctrl + A.

Why Apply Scale/Rotation (Ctrl + A)?
  • When you scale or rotate in Object Mode, Blender stores it as a transform offset, not in the mesh itself.
If you don’t apply:
  • Bevels and modifiers may look uneven (scale not uniform).
  • Physics simulations (cloth, rigid body) can break.
  • Texture mapping and shading may stretch or misalign.
  • Rigging and animation can deform incorrectly.
Rule: once an object is sized and oriented correctly, always Ctrl + A → Scale.

Real-world scale
  • 1 Blender Unit = 1 meter.
  • Tabletop: ~1.20m × 0.75m × 0.03m. Table height: ~0.75m.

Why scale matters:
  • Lighting (physically based falloff assumes meters).
  • Textures/UVs (consistent texel density).
  • Simulations (cloth, physics).
  • Exporting to other programs.

Collections & naming conventions
Collections = groups of objects in the Outliner (folders).
  • Example: a “Table” collection holding Table_Top and Legs, inside a “Props” collection.
  • Benefits: keeps scenes organized, allows selective visibility, makes exporting clean.
  • Naming objects: never leave defaults like “Cube.003.” Use clear names like Table_Top, Leg_Master, Leg_FR.
  • Why it matters: large productions can have thousands of objects. Clear names make collaboration, debugging, and referencing manageable.
  • Hotkey: Select your object and press M. Add and name your collection.

Hands-on: Tabletop
  • Delete default cube.
  • Add cube (Shift + A → Mesh → Cube). Rename to Table_Top.
  • Scale the tabletop so that it is thin and rectangular
  • Position tabletop center at around Z=0.8m.
  • Apply Scale (Ctrl + A).
Duplicates vs Linked Duplicates
Duplicate (Shift + D): creates an independent copy. You can freely modify it; changes won’t affect the original.

Linked Duplicate (Alt + D): instances that share the same mesh data. Edit one = all update. Transformations (location, scale, rotation) remain independent.

Why it’s important:
  • Saves memory and file size in complex projects.
  • Guarantees unity and consistency across repeating elements (perfect for chairs, table legs, railings, etc.).
  • Fits real studio workflows where one mesh asset is instanced across a scene.

Selecting multiple objects:
  • (Shift + Select): Selects multiple objects through selection
  • B: Creates a box selection

Hands-on: Table Legs:
  • Add a cube → Alt + D → move. Edit one = all linked duplicates update.
Edit Mode: Building the Table
What Edit Mode Is
Object Mode treats a model as a single whole unit. You move, rotate, and scale the “container.”
Edit Mode lets you go inside that object to manipulate the geometry that defines its shape:
  • Vertices (points in space).
  • Edges (lines connecting vertices).
  • Faces (polygons defined by edges).
Every 3D model is built from these components. If Object Mode is moving a Lego brick, Edit Mode is carving, reshaping, or snapping the studs on that brick.

Core Transformations in Edit Mode
The same three tools from Object Mode apply inside Edit Mode, but now they act on parts of a mesh instead of the whole object:
  • Grab/Move (G): reposition vertices, edges, or faces. Example: adjust one leg inward for proportion.
  • Rotate (R): rotate selected geometry, e.g., angle the tabletop slightly or tilt a set of extruded parts.
  • Scale (S): resize parts of the mesh. Example: taper the bottom of the table legs inward for style and stability.
These parallel transformations make Blender feel consistent: what you learned in Object Mode now applies to geometry-level edits.

Selection Modes
  • Vertex (1): precise control.
  • Edge (2): structural flow.
  • Face (3): big area manipulation.
  • Loop select: Alt + Click.
  • Connected geometry: Ctrl + L.

The Core Four Maneuvers

Loop Cut (Ctrl + R):
Adds a new line of edges (a “loop”) around your mesh.
How to use it:
  • Press Ctrl + R and hover your mouse over the object. You’ll see a preview line.
  • Use your scroll wheel to increase or decrease the number of cuts.
  • Left-click to confirm placement, then move the loop into position, or right-click to leave it centered.
Extra controls: Look at the bottom-left menu (appears after you place the cut). You can adjust the number of cuts, smoothness, or slide amount precisely.

Bevel (Ctrl + B):
Rounds or chamfers the edges of your model.
How to use it:
  • Select an edge or multiple edges.
  • Press Ctrl + B and drag your mouse to create a bevel.
  • Use the scroll wheel to increase or decrease the number of segments (more segments = smoother round).
Extra controls: In the bottom-left menu, you can set the width, segments, and even use a profile curve for custom bevel shapes.

Inset (I):
Creates a new face inside a face, like drawing a smaller rectangle within a bigger one.
How to use it:
  • Select a face.
  • Press I and drag to create an inset.
Extra controls: In the bottom-left menu, you can adjust thickness, boundary, or even inset multiple faces at once.

Extrude (E):
Pulls out new geometry from your selection to add thickness or volume.
How to use it:
  • Select a face, edge, or vertex.
  • Press E and drag in a direction.
  • Add axis constraints (X, Y, Z) if needed.
Extra controls: The bottom-left menu lets you adjust how far you extrude or set exact values.

Finish
  • Bevel tabletop edges slightly.
  • Right-click → Shade Auto Smooth.
