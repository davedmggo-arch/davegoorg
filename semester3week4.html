What You'll Learn

  • How to build and modify materials using the Material Editor (Base Color, Roughness, Metallic, Emission, and Transmission).
  • How to use procedural textures sparingly to add detail (Noise, Voronoi) without extra geometry.
  • How to set up lighting to make materials look convincing (3-point setup, HDRI, and light properties).
  • How to render scenes using Eevee and Cycles, including sampling, denoising, and output formats.
  • How to balance speed vs quality in materials, lighting, and rendering to achieve professional-looking results.

Key Concepts & Tools


Material Building
Principled BSDF Shader
Universal shader for realism and NPR.

  • Base Color: Surface albedo (wood, grass, skin tones).
  • Roughness & Metallic: Defines matte vs glossy, metal vs non-metal.
  • Specular: how much light reflects at glancing angles.
  • Transmission: Controls how much light passes through a material (transparency).
  • Emission: For glowing effects.
  • Normal/Bump Maps: Fine surface details without extra geometry.
  • Procedural Textures (basic use)
  • Noise: Random irregularity (stone, dirt).
  • Voronoi: Cells and sci-fi patterns.

Naming Conventions:
  • Don’t leave materials as “Material.001.” Use clear names: MTL_Wood_Oak_Dark, MTL_Metal_Steel_Brushed, MTL_Plastic_Red_Glossy.
  • Prefixing helps: e.g., MTL_ for materials (MTL_Glass_Window).
  • Reuse Materials: If multiple objects share the same look (e.g., all chrome parts), use linked materials instead of duplicating.

Tips & Tricks
  • Start simple: Base Color + Roughness. Add complexity only when needed.
  • Use ColorRamp nodes to fine-tune procedural textures.
  • Avoid overusing Emission — can wash out lighting if too strong.
Common Material Values (Principled BSDF starting points)
Wood
  • Base Color: RGB ~ (0.40, 0.26, 0.12)
  • Roughness: 0.6–0.8 (matte finish)
  • Metallic: 0.0
  • Add subtle Noise to Roughness + Bump for grain variation

Plastic (shiny)
  • Base Color: Any color (often bright, saturated)
  • Roughness: 0.3–0.4
  • Metallic: 0.0
  • Specular: 0.5–0.6

Metal (steel)
  • Base Color: RGB ~ (0.55, 0.55, 0.55)
  • Roughness: 0.2–0.3
  • Metallic: 1.0

Glass
  • Base Color: RGB ~ (1.0, 1.0, 1.0)
  • Roughness: 0.0–0.05
  • Transmission: 1.0
  • IOR: 1.45
  • Metallic: 0.0

Water
  • Base Color: RGB ~ (0.0, 0.05, 0.1) (slightly tinted blue)
  • Roughness: 0.0–0.05
  • Transmission: 1.0
  • IOR: 1.33
  • Add subtle Noise texture for ripples

Organic Skin Surface
  • Base Color: RGB ~ (0.9, 0.75, 0.7)
  • Roughness: 0.6–0.7 (soft, matte)
  • Subsurface: 0.3–0.4 (for translucency)
Lighting
Three Core Lighting Scenarios

Three-Point Lighting (Studio Setup)
  • Key Light: Main directional source, defines shape.
  • Fill Light: Softens shadows, adds balance.
  • Rim/Back Light: Separates subject from background.
  • Why it matters: Standard in film, photography, and CG for clear readability. Popular because it works for almost any subject.

HDRI Lighting (Environment Map)
  • Uses a 360° image for realistic ambient light.
  • Automatically adds global reflections and color.
  • Why it matters: Industry standard for product visualization, realism, and fast setup. Popular because it gives natural context lighting without much setup.

Single Light Dramatic Setup
  • One light only (often a Spot or Area).
  • Creates strong contrast and mood (like film noir or stage lighting).
  • Why it matters: Useful for storytelling, character close-ups, or design-focused renders. Popular in concept art and stylized work.

Tips & Tricks
  • Use warm key lights + cool rim lights for contrast.
  • HDRI is great for realism, but mix in extra lights for control.
  • Turn on Viewport Render Preview (Z → Rendered) to see real-time lighting feedback.
Camera Control & Settings
Camera Movement
  • Numpad 0: Switch to camera view.
  • G (Move), R (Rotate): Move camera like any object.
  • Lock Camera to View (N-Panel → View Tab):
  • Toggle this option, then move your viewport as normal (Middle Mouse drag, Shift + MMB pan, scroll zoom).
  • The camera will follow your movements — great for quickly framing shots.
  • Turn it off once the angle is set, to avoid accidentally shifting the camera.

Camera Properties

Focal Length
  • Default = 50mm.
  • Lower (<30mm) = wide-angle, more dramatic perspective.
  • Higher (>80mm) = zoom lens, flattens perspective.
  • Tip: Use 35–70mm for most scenes for realism.

  • Naming Cameras
  • Label them based on purpose, not just “Camera”:
  • CAM_FinalRender, CAM_Closeup, CAM_WideShot.
  • Multiple Cameras: Use several cameras for different compositions, then choose the active one with Ctrl + Numpad 0.
  • Viewport Lock: Use Lock Camera to View when framing shots, but turn it off after setting the angle to avoid accidental shifts.

Clip Start & End:
  • Clip Start: How close objects can be before disappearing. (Set to 0.1–0.3m for close-ups.)
  • Clip End: How far the camera can see (set higher for big environments).
Rendering
  • Engines: Eevee (fast preview), Cycles (realistic).
  • Sampling: Drafts = 32–64, Finals = 256–512+.
  • Denoising: Removes grain, especially in Cycles.
  • Color Management: Filmic for realism, Standard for flat/NPR.
  • Output Formats: PNG for transparency, EXR for compositing, JPEG for quick previews.

PNG as the Standard for This Project
  • Small file size, quick to save and share.
  • Allows for transparency
  • Universal format (every device and system can open).
  • Good balance of quality and practicality.
  • Reminder: Use high enough resolution (1920×1080 or above) so details are clear, even if PNG compresses slightly.
